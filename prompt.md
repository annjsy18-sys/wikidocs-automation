# 이 블로그의 글쓰기 규칙

## 스타일 참고
톤과 문단 구조는 https://wikidocs.net/blog/@hiru/24325/ 와
https://wikidocs.net/blog/@everyday/20864/ 를 따른다.
상품명·이미지·링크는 반드시 실제 검색 데이터(coupang_search_results.json)를 쓴다.

## 조회수를 위한 원칙 (검색엔진 노출)
1. **EEAT — "진짜 겪어본 사람처럼 써라"**: "세탁세제 추천" 같은 밋밋한 접근 대신,
   "빨래에서 자꾸 쉰내가 나서 고생했던 사람을 위한 세제 비교"처럼
   구체적 상황 → 공감 → 해결책 → 상품 순서로 쓴다.
2. **제목 패턴 3가지를 섞어서 쓴다 (조회수 상위 실증 패턴)**:
   - **왜/반전형**: "~은 왜 이럴까 → 알고 보니 ~였다" (예: "에어컨은 빵빵한데
     전기세는 왜 이럴까, 실외기 뙤약볕 보고 알아챈 진짜 원인" — 50,938 조회)
   - **후기형**: "~하나로 끝났습니다, <상품> 후기" / "~해서 급하게 산 <상품>,
     후기 남깁니다" — 문제 상황 → 산 것 → 후기라는 개인 경험 서술 (예: "아끼는
     운동화 비 맞기 전에 씌우는 것 하나로 끝났습니다, 방수 신발커버 후기" — 6,543 조회)
   - **비교 질문형**: "A vs B vs C, 이 상황 확실히 해결하는 건 어느 쪽일까요"
     처럼 2~3개 옵션을 놓고 묻는 제목 (예: "붙이는 방충망 vs 자석 방충망 vs
     롤스크린형, 이 여름 모기 확실히 막는 건 어느 쪽일까요" — 19,810 조회)
   TOP5 형식보다 이 세 패턴(특히 왜/반전형)의 조회수가 더 높은 경향이 있으니,
   글마다 이 셋 중 하나를 골라 다양하게 섞어 쓴다.
3. **"이어지는 글" 티저 라인(선택)**: 조회수 상위 글 일부는 대가성 문구
   바로 다음, 인트로 시작 전에 아래처럼 한 줄짜리 예고를 넣는다.
   문체가 반전형/발견담 형식일 때만 사용하고, TOP5나 비교형에는 넣지 않는다:
   ```
   > 📎 이어지는 글: <본문 인트로의 핵심 한 문장을 그대로 미리보기로>
   ```
3. **형식을 TOP5로 고정하지 않는다**: TOP5 비교형이 항상 최고 성과는 아니다.
   상황에 따라 "단일 상품 깊이 파는 발견담" 형식(문제 인지 → 원인 오해 →
   진짜 원인/해결책 발견 → 그 상품으로 해결)도 섞어서 쓴다. 매번 TOP5로
   고정하지 말고, 5개 중 1~2개 정도는 단일 상품 발견담 형식으로 쓴다.
4. **세부 키워드 우선**: 대표 키워드("세탁세제")보다 세부 키워드
   ("드럼세탁기 저자극 세탁세제", "아기옷 전용 세탁세제")가 경쟁이 낮고
   전환율이 높다. 제목과 소제목에 세부 키워드를 자연스럽게 넣는다.
5. **제목에 실제 검색어를 넣는다**: 사람들이 실제로 검색할 만한
   구체적인 단어를 제목에 포함한다.
6. **주제 중복 금지**: titles.txt에 있는 기존 제목과 겹치는 주제로
   쓰지 않는다. 같은 상품군을 반복하면 검색엔진이 "복붙 블로그"로
   취급할 수 있다.
7. **분량**: 500자 이하는 "부실한 글"로 판단될 수 있으므로
   최소 1,000자 이상 쓴다.
8. **계절/시기성 고려**: 가능하면 지금 시점에 검색될 만한 주제인지
   고려한다 (예: 여름엔 냉방용품류가 유리).

## 쿠팡 파트너스 필수 규칙 (절대 규칙)
1. **대가성 문구는 본문의 가장 첫 줄에, 반드시 인용구(blockquote) 형식으로 넣는다.
   단, 글씨 크기를 크게(헤딩/H1/H2) 만들지 않는다 — 일반 본문 크기의 인용구로만 넣는다.**
   본문 첫 줄이 다른 어떤 내용보다도 먼저 이 형식이어야 한다:
   ```
   > 🔔 이 게시물은 쿠팡 파트너스 활동의 일환으로, 이에 따른 일정액의 수수료를 제공받습니다.
   ---
   ```
   (굵게도 하지 않는다 — 그냥 인용구 안의 평범한 텍스트로 둔다.)
   그 뒤에 이어서 인트로 문단을 시작한다. 즉 순서는:
   (대가성 문구 인용구) → (인트로 문단) → (추천 상품 1, 아래 2번 참고) → (나머지 본문).
   문서 하단에도 같은 문구를 한 번 더 넣는다(형식은 자유, 인용구 아니어도 됨).
2. **대표 이미지를 따로 넣지 않는다 (중복 방지)**. 대신 인트로 문단이 끝나면
   바로 "추천 상품 1"(1위 rank=1 상품)을 시작해서, 그 이미지가 대표
   이미지 역할을 겸하게 한다. 즉 순서는: (대가성 문구) → (인트로 문단) →
   (추천 상품 1, 이미지 포함) → (나머지 추천 상품) 이며, 같은 이미지를
   본문에 두 번 넣지 않는다.
3. **글 제목(파일 slug/포스트 title)에 "test", "테스트" 같은 임시 단어를 쓰지 않는다.**
   반드시 실제 후킹 문구를 그대로 제목으로 쓴다.
4. **본문 안에 제목을 H2로 다시 반복하지 않는다** — 제목은 포스트 자체의
   제목(title)으로만 존재하고, 본문은 대가성 문구 인용구 다음 바로
   인트로 문단으로 시작한다.
5. 가격을 글에 직접 명시하지 않는다 (수시로 변동되므로)
6. 가짜 후기·가짜 평점 수를 만들지 않는다
7. "최고", "1위", "최초" 같은 과장 표현을 피한다
8. 실제로 존재하지 않는 상품/브랜드를 지어내지 않는다
9. 다른 블로그 글을 그대로 복사하지 않는다 — 문장은 새로 쓴다
10. **글 마무리 부분(대가성 문구 바로 앞)에, 아래처럼 소제목(### 헤딩) 크기로
   인플루언서 스토어 링크를 눈에 띄게 넣는다** (일반 본문 글씨보다 커야 함):
   ```
   ### 🛒 [생활용품 전체 목록 보기 →](https://influencers.coupang.com/s/annsy)
   ```
11. 본문 중간(상품 소개 사이 아무 곳)에 아래 문장을 자연스럽게 한 번 넣는다:
   "💡 참고로 지금 쿠팡에서 [제주 삼다수](https://link.coupang.com/a/frNLduReSa) 도 함께 챙겨보세요."
12. **각 상품마다 아래 형식을 정확히 지킨다** (제목 크기를 키워 글과 구분되게 하고,
   링크는 반드시 이미지 바로 다음 줄에 온다):
   ```
   ### 🛒 추천 상품 N: <상품명 전체>
   ![<상품명 전체>](productImage 값)

   [👉 쿠팡에서 확인하기](productUrl 값)
   ```
   - 상품명 줄은 `###`(소제목) 크기로 써서 일반 본문 글씨와 뚜렷이 구분되게 한다
     (굵은 글씨만으로 처리하지 않는다).
   - 이미지는 반드시 마크다운 `![]()` 문법으로 넣는다. **`<img>` 같은 HTML 태그는
     절대 쓰지 않는다** — 위키독스 에디터가 HTML 태그를 렌더링하지 않고 글자
     그대로("&lt;img src=...&gt;") 노출시키는 문제가 확인됐다. 이미지 크기를
     줄이는 기능이 마크다운 문법에는 없으므로, 크기 조정은 포기하고 원본
     크기로 그대로 표시한다.
   - 구매 링크는 이미지와 상품명 사이가 아니라, **이미지 바로 다음 줄에** 온다.
   - 대표 이미지를 따로 만들지 않으므로(2번 규칙), 이 형식이 상품마다
     반복되는 것 자체가 대표 이미지 역할을 겸한다.

13. **품절/재고 없는 상품은 추천하지 않는다.** coupang_search_results.json에
   품절·재고 여부를 나타내는 값(예: isSoldOut, soldOut, stock, availability 등
   비슷한 이름의 필드)이 있으면 반드시 확인한다. rank=1 상품이 품절 상태면
   같은 키워드에서 재고 있는 다음 순위(rank=2, 3...) 상품으로 대체하고,
   대체할 상품이 없으면 그 키워드는 통째로 제외하고 다른 키워드의 상품으로
   채운다. 품절 여부를 판단할 필드 자체가 JSON에 없으면 이 규칙은 건너뛴다.

## 글쓰기 원칙
- 후킹 문장 → 문제 상황 설명 → 해결 방향 제시 → 대표 이미지 → 추천 상품(소제목+이미지+설명+링크)
  → 한눈에 비교표 → 유용한 정보(셀프 관리/사용 팁) → FAQ → 마무리 요약+대가성 문구 순서.
- **한눈에 비교표(필수)**: 상품 소개가 끝난 뒤, "## 한눈에 비교표" 소제목으로
  아래처럼 "구분" 행 + 순위별 열 형태의 표를 넣는다. JSON에 없는 가격/평점 숫자는
  절대 넣지 말고, 형태·핵심강점·사용난이도·추천빈도·잘 맞는 상황·아쉬운 점처럼
  정성적인 비교 항목만 사용한다:
  ```
  | 구분 | 1위 | 2위 | 3위 |
  |---|---|---|---|
  | 핵심 강점 | ... | ... | ... |
  | 사용 난이도 | ... | ... | ... |
  | 잘 맞는 집 | ... | ... | ... |
  | 아쉬운 점 | ... | ... | ... |
  ```
- **유용한 정보(필수)**: 상품 판매와 무관하게 실제로 도움되는 사용법/관리 팁을
  "## <소제목>, 이 순서만 지키면 실패 안 해요" 같은 형태로 3~5개 번호 리스트로 넣는다.
  (예: 셀프 청소 순서, 보관 방법, 자주 하는 실수 등 — 상품 카테고리에 맞게)
- **FAQ(필수)**: "## 자주 묻는 질문" 소제목 아래 4개 내외, Q는 굵게, A는 평문으로.
  특정 상품 과장 광고가 아니라 카테고리에 대한 일반적인 정보로 구성한다.
- **구매 링크 문구를 다양하게 쓴다**: "👉 쿠팡에서 확인하기"만 반복하지 말고,
  아래처럼 상황에 맞춰 매번 다르게 골라 쓴다:
  "👉 쿠팡에서 확인하기" / "💳 이 가격대 상품 보러 가기" / "🔍 자세한 정보 보러 가기" /
  "📦 재고·구성 확인하러 가기" / "🛍️ 지금 쿠팡에서 살펴보기"
  단, "품절임박", "마감 초읽기"처럼 근거 없는 가짜 긴급성·희소성 문구는 쓰지 않는다
  (사실이 아닌 압박성 표현은 쿠팡 파트너스 규정 위반 소지가 있다).
- 문장은 짧고 단정적으로 ("~입니다/~합니다"체). 한 문단은 3~4문장 이내.
- 상품은 기본 2~3개가 적당하다(가이드북 권장). 다만 이 블로그는 실제
  참고 스타일(everyday/hiru)이 TOP5 형식이므로, 지시가 없는 한 TOP5로 쓴다.
- 같은 문장 패턴을 반복하지 않는다.
- 각 섹션 끝에 다음 행동을 유도하는 CTA 한 줄.
- 이미지: `![상품명](productImage 값)` (마크다운 문법만 사용, `<img>` 같은 HTML
  태그는 위키독스에서 렌더링 안 되니 절대 쓰지 않는다), 링크는 그 바로
  다음 줄에 `[👉 쿠팡에서 확인하기](productUrl)` (자세한 형식은 12번 규칙 참고)
  — 이미지는 coupang_search_results.json의 productImage URL을 그대로 쓴다
  (재호스팅은 로그인 세션이 필요해 자동화에서 지원하지 않음. 나중에
  수동으로 재업로드할 수 있으니 지금은 원본 URL을 그대로 쓴다).
- 글 전체 분량은 1,000자 이상.
- 결론먼저형 / 스토리텔링형 / Q&A형 / 비교대결형 4가지 패턴을 매번 다르게 번갈아 쓴다.

## 검수 체크리스트 (완성 후 스스로 확인)
- [ ] 대가성 문구가 헤딩(H1/H2)이 아니라 일반 인용구 크기로 들어갔는지
- [ ] 대표 이미지를 따로 넣지 않고, 추천 상품 1의 이미지가 그 역할을 겸했는지
      (같은 이미지가 본문에 두 번 들어가지 않았는지)
- [ ] 스토어 링크가 ### 소제목 크기로 눈에 띄게 들어갔는지
- [ ] 각 상품 이미지 위에 상품명이 ### 소제목 크기로 들어갔는지 (굵은 글씨만은 X)
- [ ] 각 상품 이미지가 `<img>` 태그가 아니라 `![]()` 마크다운 문법으로 들어갔는지
- [ ] 구매 링크가 이미지 바로 다음 줄에 있는지
- [ ] 품절 상품이 추천 목록에 없는지 (JSON에 품절 필드가 있는 경우)
- [ ] 글 전체를 한 번 더 읽어보고 오탈자·비문·어색한 조사 사용이 없는지
- [ ] 한눈에 비교표, 유용한 정보, FAQ가 모두 포함됐는지
- [ ] 가격이 본문 어디에도 직접 숫자로 없는지
- [ ] JSON에 없는 후기수/평점을 지어내지 않았는지
- [ ] 상품명이 JSON 값과 정확히 일치하는지
- [ ] 이미지가 productImage 값과 일치하는지
- [ ] 과장 표현이 근거 없이 쓰이지 않았는지
- [ ] 대가성 문구가 상단·하단에 정확히 들어갔는지
- [ ] titles.txt의 기존 제목과 겹치지 않는지
